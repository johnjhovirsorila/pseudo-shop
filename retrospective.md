## 3. Describe a merge conflict you encountered. What caused it and how did you resolve it?
The group encountered a merge conflict when two developers worked on different features and added in the same file (store_view.pseudo). To further expound, this was due to the fact that both users started with the skeleton file and inputted their respective features in individual branches. When the team lead merged the requests from both developers, it displayed a conflict as the members wanted to edit the same file. Ultimately, this was resolved by the team lead by clicking the "accept both changes" in the GitHub platform due to each feature having no redundancy and logical error by the members.

## 4. What were the biggest challenges you faced as a team?
As the activity was performed and completed in a remote setup, one of the biggest challenges encountered by the group is communication and coordination. The group had to ensure that little to no merge conflicts occur, and if conflicts do occur, the group has to agree on which branches are to be pushed into the main branch. Although this is a normal situation, the time wasted for added communication could have been avoided.

Focusing on the team lead, he was able to learn how to fork from the main branch and how to detect/solve any merge conflicts that may occur in the future. Lastly, all of these activities and findings were also shared to different members of the group through an online Google Meet.
